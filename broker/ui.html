Guide — installation & mises à jour

Ce broker : . Les commandes se lancent depuis le dossier mailbox-handoff (PowerShell). Remplace les <…>.

🖥️ Serveur (le broker) — sur la machine qui héberge la messagerie

A. Première installation

  1. Copie/clone le dossier mailbox-handoff sur la machine serveur.
  2. Dans un PowerShell Administrateur : npm install (installe better-sqlite3, requis côté broker).
  3. Lance le broker — au choix :
.\setup-server.ps1            # broker en avant-plan (test)
.\setup-server.ps1 -Service   # VRAI service Windows (auto-démarrage, sans session)

B. Mise à jour vers une nouvelle version

  1. git pull (ou recopie le dossier) sur la machine serveur.
  2. npm install (si de nouvelles dépendances).
  3. Arrête l'ancien broker — ferme sa fenêtre, ou .\setup-server.ps1 -RemoveService si c'était un service.
  4. Relance : .\setup-server.ps1 -Service.

↪ Au 1er démarrage de la nouvelle version, l'ancien store.json est migré automatiquement en base SQLite (store.db) — tes messages et canaux sont conservés.

C. Installer le service depuis cette page — onglet Serveur : possible uniquement si tu ouvres l'UI sur la machine serveur (http://localhost:<port>/) et que le broker a été lancé en PowerShell Admin.

💻 Client (chaque projet / machine qui dialogue)

A. Rattacher un projet — sur la machine cliente :

.\setup-client.ps1 -Project <nom> -Broker http://<ip>:7777 -ProjectDir <chemin-du-projet> -Channels <canal>

L'onglet Config client génère cette commande remplie pour toi. Puis recharge la session Claude Code du projet. Important : repasse -Channels à chaque réinstall, sinon l'abonnement au canal est perdu.

B. Mettre à jour un client (et corriger la réception) :

.\install.ps1 -SkipProject

Réécrit les scripts ~/.claude/ + corrige le hook (chemin absolu, dédoublonné). À faire si « je dois dire à l'agent d'aller voir sa boîte pour qu'il voie le message » : c'est le signe d'un hook branché en %USERPROFILE% non expansé.

📡 Canaux   ·   🧵 Fils (question/réponse)

  • S'abonner à un canal (ex. sanofi-ls1) : ajoute -Channels sanofi-ls1 à l'install client. Les messages du canal sont conservés côté broker quoi qu'il arrive.
  • Écrire à un canal : /msg #sanofi-ls1 <texte> — seuls les abonnés le reçoivent.
  • Répondre dans un fil : tool MCP mailbox_reply (replyTo=<id>) ou /msg --reply <id> <texte>. Relire un fil : mailbox_thread ou l'onglet Fils.
